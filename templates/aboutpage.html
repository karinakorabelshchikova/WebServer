Как работает этот сайт?


Что планировалось


• Тема проекта:
Сайт для любителей архитектуры. Возможность посмотреть на здания университетов со всего мира.
• Техническое задание:
Сайт со множеством разделов. В их числе: Главная, О сайте и Страница университета (шаблон, на основе которого генерируются страницы для каждого из университетов). Сайт объединяет дизайн, в том числе шапка и подвал (с функциональными кнопками). Информация для сайта храниться в базе данных.


Что получилось


Три составляющие сайта:

  • main.py - програмная часть
  • universities.sqlite - все данные об университетах
  • папки static и templates - статический контент (иконки и макеты страниц)

Во время создания сайта постоянно возникали каверзные вопросы. На некоторые из них даже удалось найти ответы. Вот некоторые из них:

  • Как можно передать картинку из базы данных, если HTML принимает только URL? Ответ: через URL. Потом и кровью.
  • Как можно передать случайную картинку из базы данных, если такая идея возникла в середине проекта? Ответ: написан свой декоратор.
  • Что делать, если sqlite не дружит с регистрами кириллицы? Ответ: добавить ещё столбец в таблицу, потому что python - дружит.
